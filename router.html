基本的操作方法
コンソールケーブルでPCとつなぎ、TeraTermなどのターミナルソフトで操作する
応用的操作方法
ルータ／スイッチでリモートログイン（Telnetやssh）を許可し、TeraTermなどのターミナルソフトでネットワーク経由で接続して操作する
コマンド入力のコツ
  • 操作はすべてコマンド入力（場合によりオプションやパラメータも追加）により行う
  • 多くのコマンドやオプションは省略形でも入力できる　例：configure terminal → conf t
  • コマンドやオプション文字列を途中まで入力してTabキーを押すと補完する（入力部分のみでそのコマンドやオプションが特定できる場合）
  • コマンド入力中に ? キーを押すと、そのうしろに入力可能なオプションが表示される
主な設定モード
ユーザーモード、特権モード、グローバルコンフィギュレーションモード、インターフェイスコンフィギュレーションモードなどがある
ユーザーモード
ほとんど何も操作できない → enable または en で特権モードへ移行
特権モード
各種設定の確認のみできる → configure terminal または conf t でグローバルコンフィギュレーションモードへ移行
グローバルコンフィギュレーションモード
全体的な設定が可能 → interface xx または int xx でインターフェイスコンフィギュレーションモードに移行
インターフェイスコンフィギュレーションモード
選択されたインターフェイスに対する設定が可能
戻る操作
  • exit または exi で一つ前のモードに戻る
  • end で特権モードに戻る
  • disable または disa で特権モードからユーザーモードに戻る
no
各種コンフィギュレーションモードで no (コマンド) を実行すると設定が削除される

特権モードで使用する主なコマンド

show running-config ( sh run )
現在のすべての設定を確認する
show ip interface brief ( sh ip int brie )
すべてのインターフェイスのステータスを簡易表示する
show ip route ( sh ip ro )
ルーティングテーブルを確認する
copy running-config startup-config ( copy run star )
設定した内容を保存する
erase startup-config ( era star )
初期化（＝起動時の設定を削除する）
show vlan ( sh vl )
作成したVLANを表示する（Cisco Switch）
show vlan-switch
作成したVLANを表示する（Cisco Router）
delete vlan.dat
VLANデータベース削除（Cisco Switch）
reload
再起動

グローバルコンフィギュレーションモードで使用する主なコマンド

interface fastethernet 0 ( int fa 0 )
FastEthernet 0のインターフェイスコンフィギュレーションモードに移行する
interface gigabitEthernet 1/0/1 ( int gi 1/0/1 )
GigabitEthernet 1/0/1のインターフェイスコンフィギュレーションモードに移行する
1/0/1　１番目の値：スタックメンバー番号（1台の場合は1）　２番目の値：モジュール番号（Catalyst 3750では常に0）　３番目の値：ポート番号（1から最大ポート数まで）
interface range gigabitEthernet 1/0/1 - 12
GigabitEthernet の 1/0/1 ～ 1/0/12 までまとめて指定する
interface vlan 1
VLAN1のインターフェイスコンフィギュレーションモードに移行する
ip routing
ルーティングを有効化する　※これを実行しないとルータとして機能しない
ip route [NetAddr] [SubnetMask] [NextHop]
ルーティングを設定する
hostname NAME
対象機器のホスト名をNAMEにする
enable password PASSWORD
特権パスワードをPASSWORDにする
line vty 0 4
ラインコンフィギュレーションモードに入る
→次の画面で
password VTYPASSWORD
login
transport input telnet
を実行することでVTYパスワードとして VTYPASSWORD が設定される
vlan XX
VLAN XX(XXは数字)が作成される
no vlan XX
VLAN XXを削除する

インターフェイスコンフィギュレーションモードで使用する主なコマンド

ip address [IP] [SubnetMask]
対象のインターフェイスにIPアドレスを設定する
no shutdown
対象のインターフェイスを有効化する　※Ciscoのルータのインターフェイスは起動時は無効になっているため no shutdown が必要
switchport mode access
対象のインターフェイスをアクセスモードにする
switchport access vlan XX
対象のインターフェイスをVLAN XXにマッピングする
